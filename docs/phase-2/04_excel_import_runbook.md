# Runbook import Excel Giai đoạn 2

## 1. Phạm vi

Runbook này áp dụng cho import có kiểm soát trên PostgreSQL local. Không cho phép production import, production deployment hoặc ghi credential vào command history/tài liệu. File Excel và mọi audit output phải nằm ngoài Git.

Nguồn duy nhất được chấp nhận là file, sheet và SHA-256 trong `config/phase-2/source-contract.json`. Không gõ lại danh sách header hoặc kỳ vọng dữ liệu ở script khác.

## 2. Hard gates trước import

1. Dùng Node.js theo version đã khóa của dự án.
2. Xác nhận đúng nhánh và working tree không có file cấm.
3. Khởi động PostgreSQL local và xác nhận healthy.
4. Chạy Prisma migration status; database phải up to date.
5. Xác nhận `ueb_core_data`, `import_run`, `workflow_event` đều rỗng cho initial import.
6. Xác nhận `stt` là `INTEGER GENERATED BY DEFAULT`, start `2570`, và `khoi_kien_thuc` là `INTEGER NOT NULL`.
7. Không gọi `nextval()` khi kiểm tra sequence.

Nếu bất kỳ hard gate nào sai, dừng trước import. Không reset, truncate, update, delete hoặc sửa workbook để làm gate đạt.

## 3. Source inspection

```sh
pnpm data:inspect -- \
  --file data/input/CSDLCore_chuan_hoa_PostgreSQL.xlsx \
  --sheet csdlcore
```

Kết quả phải khớp source contract: raw SHA, 20 header đúng thứ tự, 2.497 dòng, 2.497 STT distinct, 74 khoảng trống STT, không formula/error cell và không có ngày/trạng thái không hợp lệ.

Profile `khoi_kien_thuc`:

```sh
pnpm data:profile-column -- \
  --file data/input/CSDLCore_chuan_hoa_PostgreSQL.xlsx \
  --column khoi_kien_thuc
```

Phải có 2.497 safe integer, range `1`–`5`, không decimal, blank, formula, error hoặc giá trị ngoài PostgreSQL int32.

## 4. Dry-run

```sh
pnpm data:dry-run -- \
  --file data/input/CSDLCore_chuan_hoa_PostgreSQL.xlsx \
  --sheet csdlcore
```

Dry-run không kết nối database. Chỉ tiếp tục khi:

- status `PASS`;
- canonical rows `2497`;
- contract violations `0`;
- dataset checksum khớp contract/bằng chứng nghiệm thu.

## 5. Controlled import

Lấy checksum từ contract thay vì gõ thủ công:

```sh
SOURCE_SHA="$(jq -r '.source_sha256' config/phase-2/source-contract.json)"

pnpm data:import -- \
  --file data/input/CSDLCore_chuan_hoa_PostgreSQL.xlsx \
  --sheet csdlcore \
  --confirm-sha "$SOURCE_SHA"
```

Import dùng migration credential tách biệt, advisory lock và một transaction. Thành công phải trả `COMMITTED`, inserted rows `2497`, đúng một `import_run`, không có workflow event và không skip/update/delete.

Không chạy lại import trước verify. Không dùng runtime credential để thực hiện controlled legacy import.

## 6. Verify độc lập

```sh
pnpm data:verify -- \
  --file data/input/CSDLCore_chuan_hoa_PostgreSQL.xlsx \
  --sheet csdlcore
```

Verify dùng runtime credential và transaction read-only. Exit code phải bằng `0`, anomalies bằng `0`; source và database dataset checksum phải trùng. Kiểm tra bổ sung:

- row count và distinct STT đều `2497`, min `-1`, max `2569`;
- đủ 20 business columns khớp từng dòng;
- mọi row checksum, deterministic ID, snapshot và provenance khớp;
- `version_no = 1`, `origin = LEGACY_IMPORT`, `source_submission_id IS NULL` cho toàn bộ dòng;
- `workflow_event = 0`;
- sequence start `2570`, chưa tiêu thụ.

## 7. Duplicate-import negative test

Chỉ chạy sau khi verify đầu tiên PASS. Chạy lại cùng command import phải trả exit code khác `0`, inserted rows `0` và failure code thể hiện core table không rỗng hoặc source đã tồn tại. Sau test, row counts, `import_run`, workflow event và dataset checksum phải không đổi.

Không xóa dữ liệu để thử import lại. Không dùng `skipDuplicates`.

## 8. Backup audit local

Có thể tạo schema-only snapshot trước import và custom-format dump sau import trong `infra/audit/phase-2`. Runtime role không có quyền đọc `_prisma_migrations`; nếu dùng runtime role, phải loại riêng bảng nội bộ này thay vì mở rộng quyền.

Dump và checksum phải được Git ignore. Bằng chứng local không đóng production backup/restore gates và không thay thế restore test.

## 9. Xử lý lỗi

- Contract violation: dừng, giữ nguyên workbook và lập báo cáo an toàn.
- Import failure: xác nhận transaction rollback, counts không đổi, không tự bỏ qua dòng.
- Database không rỗng: không import, không reset hoặc truncate.
- Checksum thay đổi: source decision và contract hiện tại mất hiệu lực; inspect và phê duyệt lại.
- Không ghi email, tên, mã cán bộ, credential, connection string hoặc đường dẫn tuyệt đối vào tài liệu/báo cáo được commit.

## 10. Production boundary

Runbook này không ủy quyền production migration/import. Production chỉ được thực hiện sau formal approval, infrastructure sign-off, backup/restore readiness và một yêu cầu triển khai riêng có thẩm quyền.
